## 项目简介

**Sudden Disaster Rescue (SDR)** 是一个综合性的突发灾害救援管理平台，旨在通过信息化手段提高灾害响应速度和救援效率。系统采用前后端分离架构，包含 Web 管理后台、微信小程序移动端以及 Java 后端服务，实现了从灾情上报、指挥调度到任务执行的全流程闭环管理。

## 系统架构

本项目包含三个核心子项目：

1.  **SDR-Vue**: Web 管理后台，供指挥中心管理员使用。
2.  **SDR-WX**: 微信小程序，供一线救援人员、志愿者和受灾群众使用。
3.  **SuddenDisasterRescue**: 后端 API 服务，为 Web 端和小程序提供数据支持。

## 功能模块

### 1. 综合指挥调度 (Web 端)
-   **数据大屏**: 基于 ECharts 和高德地图的灾情可视化大屏。
-   **灾情管理**: 灾害事件的录入、审核与分级。
-   **资源管理**: 救援队伍、物资仓库、避难场所的维护。
-   **任务调度**: 指挥中心向救援队指派任务，实时跟踪任务进度。
-   **用户权限**: 基于角色的权限控制 (RBAC)，管理管理员、救援队和志愿者。

### 2. 移动救援协同 (小程序端)
-   **一键上报**: 群众或救援人员现场上报灾情 (含定位、图片)。
-   **任务接收**: 救援队员接收指派任务，更新执行状态。
-   **紧急通知**: 接收系统推送的预警信息和调度指令。
-   **个人中心**: 志愿者注册、身份认证与信息管理。

### 3. 核心服务支撑 (后端)
-   **多端认证**: 支持账号密码、短信验证码、微信一键登录。
-   **消息中心**: 集成 RabbitMQ 实现异步消息解耦，支持 WebSocket 推送。
-   **安全防护**: 统一的全局异常处理、参数校验与鉴权机制。

## 环境依赖

在运行本项目之前，请确保您的开发环境满足以下要求：

-   **Java**: JDK 1.8
-   **Node.js**: v20.19.0 或更高版本 (推荐 v22)
-   **Database**: MySQL 8.0+
-   **Cache**: Redis 5.0+
-   **Message Queue**: RabbitMQ (可选，若不启用需注释相关配置)
-   **Maven**: 3.6+

## 快速上手

本项目分为三个独立部分，请分别参照各子目录下的 `README.md` 进行配置与启动：

1.  **后端启动**:
    -   进入 `SuddenDisasterRescue/`
    -   配置 `application.yml` (数据库、Redis)
    -   运行 `SDRApplication.java`

2.  **管理后台启动**:
    -   进入 `SDR-Vue/`
    -   运行 `npm install` && `npm run dev`

3.  **小程序启动**:
    -   使用 **微信开发者工具** 导入 `SDR-WX/` 目录
    -   配置 AppID 并编译预览
